# 云健健身小程序

基于 uniapp 开发的健身管理小程序，对应 CloudFitness 项目的移动端应用。

## 项目结构

```
user/cloudfit/
├── api/                    # API 接口文件
│   ├── auth.js           # 认证相关接口
│   ├── course.js         # 课程相关接口
│   ├── reservation.js    # 预约相关接口
│   ├── locker.js         # 储物柜相关接口
│   ├── attendance.js     # 签到相关接口
│   ├── membership.js     # 会员卡相关接口
│   └── user.js           # 用户相关接口
├── pages/                 # 页面文件
│   ├── index/            # 首页
│   ├── login/            # 登录页
│   ├── register/         # 注册页
│   ├── courses/          # 课程中心
│   ├── reservations/     # 我的预约
│   ├── lockers/          # 储物柜
│   ├── attendance/       # 签到记录
│   └── profile/          # 个人中心
├── store/                 # 状态管理
│   └── user.js           # 用户状态管理
├── utils/                 # 工具函数
│   ├── request.js        # 请求封装
│   └── date.js           # 日期工具
├── static/                # 静态资源
├── App.vue                # 应用入口
├── main.js                # 主入口文件
└── pages.json             # 页面配置

```

## 技术特点

1. **MVP 实现**：核心功能完整，界面简洁实用
2. **状态管理**：使用简单的 store 模式管理用户状态
3. **API 封装**：统一的请求封装，自动处理 token 和错误
4. **响应式设计**：适配不同屏幕尺寸
5. **用户体验**：加载状态、错误提示、操作反馈完善

## 配置说明

### API 地址配置

在 `utils/request.js` 中修改 `BASE_URL`：

```javascript
const BASE_URL = 'http://localhost:8080/api' // 根据实际情况修改
```

### TabBar 图标

需要在 `static/` 目录下添加以下图标文件（或修改 `pages.json` 中的图标路径）：
- `home.png` / `home-active.png` - 首页图标
- `course.png` / `course-active.png` - 课程图标
- `profile.png` / `profile-active.png` - 个人中心图标

如果暂时没有图标，可以注释掉 `pages.json` 中的 `tabBar` 配置，使用纯文字导航。

## 运行说明

1. **开发环境**：
   - 使用 HBuilderX 打开项目
   - 配置后端 API 地址
   - 运行到微信开发者工具或其他平台

2. **API 对接**：
   - 确保后端服务已启动（默认端口 8080）
   - 修改 `utils/request.js` 中的 `BASE_URL` 为实际后端地址
   - 如果后端部署在不同域名，需要配置跨域

3. **登录流程**：
   - 首次使用需要注册账号
   - 注册成功后跳转到登录页
   - 登录成功后自动跳转到首页

## 注意事项

1. **MVP 版本**：当前为 MVP（最小可行产品）版本，功能已实现但可能不够完善
2. **错误处理**：部分错误处理较为简单，生产环境需要完善
3. **数据验证**：前端验证已实现，但建议后端也要做相应验证
4. **安全性**：Token 存储在本地存储，生产环境建议使用更安全的存储方式
5. **图标资源**：TabBar 图标需要自行添加，或使用文字导航
